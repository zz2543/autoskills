# 新任务实验运行指南

本文描述在 APO-SkillsMD 框架中，从零开始对一个新任务跑 skill 演化实验的完整操作流程。

---

## 1. 前置条件

### 1.1 环境安装

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .[dev]
```

### 1.2 API Key 配置

复制模板并填写实际使用的 provider key：

```bash
cp .env.example .env
```

`.env` 内容示例（只填你实际用到的）：

```bash
# OpenAI-compatible 代理（当前默认）
OPENAI_API_KEY=sk-xxxxx

# MiniMax
MINIMAX_API_KEY=xxxxx
MINIMAX_BASE_URL=https://api.minimax.io/v1
MINIMAX_MODEL=MiniMax-M2.1
```

### 1.3 验证连通性

用本地 mock 跑一遍烟测，确认框架本身没问题：

```bash
python scripts/run_experiment.py \
  --config configs/local_mock.yaml \
  --task-path tests/fixtures/mock_task.json \
  --out results/smoke_check
```

`results/smoke_check/experiment_summary.json` 里出现 `best_pass_rate` 字段即说明流程通。

## 2. 第一步：准备任务文件

在 `data/skillsbench/tasks/` 下创建任务的 JSON 文件。

### 2.1 文件格式

```json
{
  "task_id": "your-task-name",
  "domain": "your-domain",
  "description": "完整描述 agent 需要完成什么，越具体越好。",
  "inputs": {
    "key": "value"
  },
  "verifier": {
    "command": "python verify.py",
    "expected_file": "output/result.csv",
    "expected_stdout_contains": "SUCCESS"
  },
  "test_cases": [
    {
      "case_id": "case-1",
      "input_payload": {},
      "expected_output": {
        "content": "精确的期望文件内容"
      }
    },
    {
      "case_id": "case-2",
      "input_payload": {"param": "value2"},
      "expected_output": {}
    }
  ]
}
```

### 2.2 字段说明

| 字段 | 是否必须 | 说明 |
|------|----------|------|
| `task_id` | 必须 | 唯一标识，用于结果目录命名（避免 `/` 以外的特殊字符） |
| `domain` | 建议填 | 用于 BM25 检索时的领域权重 |
| `description` | 必须 | agent 的任务描述，也用于 LLM 生成初始 skill |
| `inputs` | 可选 | 任务输入 payload，注入到 agent 的执行上下文 |
| `verifier.command` | 推荐 | 官方验证器命令，框架会在 agent 执行后运行并检查 exit code |
| `verifier.expected_file` | 可选 | 框架检查该文件是否存在 |
| `verifier.expected_stdout_contains` | 可选 | 框架检查 agent 输出是否包含该字符串 |
| `test_cases` | 推荐 ≥3 个 | 每个 case 独立评估；pass rate = 通过数 / 总数 |

### 2.3 verifier 三种模式的组合使用

三个 verifier 字段可以同时使用，框架按顺序全部检查：

1. **`command`**（最强）：运行外部脚本，exit code 0 = pass，非零 = fail
2. **`expected_file`**：检查文件是否存在，如果 `expected_output.content` 也有值，同时校验内容
3. **`expected_stdout_contains`**：检查 agent 的最终输出是否包含指定字符串

## 3. 第二步：准备初始 Skill 池（可选）

框架会自动用 BM25 从 skill 池检索与任务相关的 skill 作为初始种群。

### 3.1 下载公开 skill 池

```bash
python scripts/download_skill_pools.py --out data/skill_pool
```

### 3.2 如果没有 skill 池

框架会调用 LLM 自动生成初始 skill，数量由 `max_llm_generated_ratio` 控制（默认最多 30% 的种群规模由 LLM 生成）。这不影响实验运行，只是初始多样性会略低。

### 3.3 手动放入已有 skill

每个 skill 是一个目录，包含 `SKILL.md` + `scripts/` 子目录：

```
data/skill_pool/
  my-existing-skill/
    SKILL.md
    scripts/
      main.py
```

`SKILL.md` 格式：

```markdown
---
id: my-existing-skill
name: My Existing Skill
description: 一句话描述这个 skill 做什么
version: 0.1.0
tags:
  - domain-tag
---

## Goal
...

## Workflow
1. ...
2. ...
```

## 4. 第三步：创建实验配置

从 `configs/default.yaml` 派生一个新配置文件。

### 4.1 最小配置示例

```yaml
# configs/my_task.yaml
extends: default.yaml
experiment_name: my_task_exp

llm:
  provider: openai
  model: gpt-5.2
  base_url: https://api.aicodemirror.com/api/codex/backend-api/codex/v1
  temperature: 0.2
  max_tokens: 2048
  use_cache: true

evolution:
  population_size: 8
  generations: 3
  retrieval_top_k: 10
  enable_trace: true
  enable_crossover: true
  enable_pareto: true
  enable_escape: false   # 当前默认关闭，escape 只注入合成种子，暂无增益

experiments:
  default_output_dir: results/my_task_exp
```

### 4.2 关键参数说明

| 参数 | 默认值 | 建议 |
|------|--------|------|
| `evolution.population_size` | 6 | 正式实验建议 ≥ 10 |
| `evolution.generations` | 2 | 正式实验建议 3～5 |
| `evolution.retrieval_top_k` | 8 | skill 池大时可调到 12 |
| `evolution.enable_escape` | false | 暂时保持关闭 |
| `llm.use_cache` | true | 实验期间建议开启，减少重复 API 调用 |
| `sandbox.max_steps` | 8 | agent 执行步数上限，复杂任务可调到 12～16 |
| `sandbox.command_timeout_sec` | 30 | 单条命令超时，重计算任务可适当提高 |

### 4.3 provider 切换

| Provider | 配置方式 |
|----------|----------|
| OpenAI-compatible 代理 | `provider: openai` + 正确的 `base_url` |
| MiniMax | `provider: minimax`，同时在 `.env` 填 `MINIMAX_*` |
| 本地 mock（不需要 key） | `provider: mock`，用于调试框架 |

## 5. 第四步：运行实验

```bash
python scripts/run_experiment.py \
  --config configs/my_task.yaml \
  --task-id your-task-name \
  --out results/my_task_exp
```

### 5.1 常用参数

| 参数 | 说明 |
|------|------|
| `--config` | YAML 配置路径（必须） |
| `--task-id` | 任务 ID，框架在 `skillsbench_dir` 下自动查找对应 JSON |
| `--task-path` | 直接指定任务 JSON 路径（与 `--task-id` 二选一） |
| `--out` | 覆盖结果输出目录 |
| `--population-size` | 临时覆盖种群大小 |
| `--generations` | 临时覆盖代数 |
| `--experiment` | 选择实验类型：`exp1_main`（默认）、`exp2_ablation`、`exp3_init`、`exp4_qualitative` |

### 5.2 示例命令

```bash
# 用任务 ID 运行（自动查找 data/skillsbench/tasks/your-task-name.json）
python scripts/run_experiment.py \
  --config configs/my_task.yaml \
  --task-id your-task-name \
  --out results/my_task_exp

# 用路径直接指定任务文件
python scripts/run_experiment.py \
  --config configs/my_task.yaml \
  --task-path data/skillsbench/tasks/your-task-name.json \
  --out results/my_task_exp

# 临时缩小规模快速验证
python scripts/run_experiment.py \
  --config configs/my_task.yaml \
  --task-id your-task-name \
  --population-size 4 \
  --generations 1 \
  --out results/quick_check
```

## 6. 结果解读

实验完成后，结果在 `results/my_task_exp/<task_id>/` 下：

```
results/my_task_exp/
  experiment_summary.json       ← 全局汇总
  experiment_summary.csv
  your-task-name/
    generation_0/
      population.jsonl          ← 初始种群评估
      mutants.jsonl             ← 第0代无变异
      offspring.jsonl
      selected.jsonl
    generation_1/
      population.jsonl          ← 上代选中的
      mutants.jsonl             ← 变异产生的候选
      offspring.jsonl           ← 交叉产生的候选
      selected.jsonl            ← 进入下一代的
      evaluations.jsonl
    ...
    final_pareto.json           ← 最终 Pareto 前沿
    summary.csv                 ← 每代 best pass rate
    safety_rejects.jsonl        ← 被安全层拦截的候选
```

### 6.1 关键指标

**`experiment_summary.json`**：
```json
{
  "experiment_name": "my_task_exp",
  "task_count": 1,
  "runs": [
    {
      "task_id": "your-task-name",
      "best_skill_id": "evolved-skill-xxx",
      "best_pass_rate": 0.85,
      "pareto_size": 4
    }
  ]
}
```

**`summary.csv`**（pass rate 曲线）：
```
generation,best_pass_rate
0,0.33
1,0.67
2,0.85
```

**`final_pareto.json`**（Pareto 前沿上的 skill，按 pass rate vs token cost 权衡）：
```json
[
  {
    "skill_id": "evolved-skill-a",
    "pass_rate": 0.85,
    "execution_tokens": 1200,
    "rank": 1
  },
  {
    "skill_id": "evolved-skill-b",
    "pass_rate": 0.70,
    "execution_tokens": 450,
    "rank": 1
  }
]
```

Pareto 前沿上的 skill 各自代表不同的 pass rate / token cost 权衡点。实验结果取用哪个，取决于部署优先精度还是成本。

### 6.2 诊断问题

| 现象 | 可能原因 | 排查方向 |
|------|----------|----------|
| `best_pass_rate` 一直是 0 | verifier 配置不对 / agent 跑完没输出 | 检查 `generation_0/population.jsonl` 里的 notes 字段 |
| `safety_rejects.jsonl` 很多 | 生成的 skill 触发了安全规则 | 检查 findings 字段，看是什么规则被触发 |
| 每代 pass rate 不涨 | 变异没有改进 / 任务描述不够具体 | 检查 `mutants.jsonl`，看 pass_rate 是否有变化 |
| LLM 调用失败 | base_url / API key 配置错误 | 用 mock provider 先验证流程，再换真实 provider |

## 7. 在任务需要特殊环境时（Docker 沙盒）

如果任务依赖特定工具（如 Trivy、特殊 Python 库），需要用 `DockerSandbox` 替换默认的 `SubprocessSandbox`。

当前框架的 `DockerSandbox` 已实现完整，但主实验流程默认仍使用 `SubprocessSandbox`。用 Docker 沙盒需要修改 evaluator 构建处（`scripts/run_experiment.py`）或使用 pilot 脚本。

**前提条件**：宿主机 Docker daemon 运行中，`docker` 命令可用：
```bash
docker info
```

**`DockerSandbox` 支持的参数**：
```python
DockerSandbox(
    profile=SandboxProfileName.OFFLINE_LOCAL,
    image="python:3.11-slim",      # 可换成任务专用镜像
    base_dir="/path/to/workspace",  # 可选，固定挂载目录
    max_output_chars=6000,
)
```

## 8. 快速问题排查清单

在报告实验不工作之前，先逐项确认：

- [ ] `.env` 里对应 provider 的 API key 已填写
- [ ] `llm.base_url` 和实际 provider/区域匹配
- [ ] 任务 JSON 的 `task_id` 和 `--task-id` 参数一致
- [ ] `verifier` 配置至少有一个有效字段（command / expected_file / expected_stdout_contains）
- [ ] `test_cases` 里的 `case_id` 唯一
- [ ] mock 烟测先通过：`python scripts/run_experiment.py --config configs/local_mock.yaml --task-path tests/fixtures/mock_task.json --out results/smoke`
- [ ] `results/<your_exp>/safety_rejects.jsonl` 里没有意外拦截

## 9. 当前框架限制与注意事项

- **`enable_escape: false`**（默认）：逃逸机制暂时关闭。escape 当前只注入合成种子 skill，对实验结果帮助有限，等 Lamarckian 反推实现后再开启。
- **Exp 2 / Exp 3 / Exp 4**：ablation、初始化对比、定性分析实验的完整协议暂未实现，当前只走 Exp 1 主流程。
- **Anthropic / Gemini / Qwen**：provider 入口已预留，但 `complete()` 是 `NotImplementedError`，切过去会报错。
- **DockerSandbox**：已实现，但主流程默认不启用，需要手动在代码里切换。
- **Lineage-guided mutation**：谱系归因分支保留轻量占位，不实现 GEPA 式归因。
